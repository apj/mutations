Helix - Software Evolution Data Set

The Helix Data Set is a compilation of release histories of a number of non-trivial Java Open Source Software Systems. It has been developed to assist researchers in the field of empirical software engineering with a focus on software evolution.

If you use the data set, do drop an email to let us know about any publications. We would like to add it into our publications section.

To cite the data set -- [BibTex,RIS,Text]

In a nut shell

  • Open Source Java software systems
  • Minimum size of each release is 100 classes(most are far larger)
  • Minimum number of releases for any software system is 15
  • Minimum release history is 18 months
  • Currently over 40 software systems, spanning 1000+ releases and over 65,000 classes
  • Evolution history of each software system is available as a ZIP file, with consistent meta-data.
  • Metric extraction tools

The data selection criteria and rationale is available in this paper -- Data Selection Criteria (PDF)

Data available

  • JAR files of each release in the evolution history
  • Range of metrics extracted for each release
  • Release notes, Defect logs for each software system

The metric extracted is discussed in this paper -- Metric Extraction (PDF)

Collaboration and Contibutions

We are keen to build strong and ongoing relationships to ensure that the data set can be kept updated regularly, and to expand the size of the data set

If you are interested, send an email to the primary maintainer of the data set - Rajesh Vasa (rvasa[at]swin.edu.au)
